# Task Tracker — Система управления задачами

## О проекте

**Task Tracker** — это веб-приложение для управления задачами в корпоративной среде. Система поддерживает трёхуровневую иерархию пользователей и позволяет эффективно распределять задачи, отслеживать их выполнение и анализировать загрузку сотрудников.

### Бизнес-задачи

- Автоматизация процессов постановки, назначения, выполнения и контроля задач
- Разделение прав доступа для разных ролей пользователей
- Управление проектами и задачами в рамках компании
- Формирование отчётов и статистики по проектам

### Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Администратор** | Полное управление системой: регистрация/увольнение сотрудников, создание проектов, назначение тимлидов |
| **Тимлид** | Управление задачами в своих проектах: создание, назначение исполнителей, контроль статусов, статистика |
| **Сотрудник** | Выполнение задач, изменение статуса (Новая → В работе → На проверке), комментарии |

## База данных

ER-диаграмма и реляционные модели храняться в Task_tracker.drawio

### Хранимые процедуры

| Процедура | Назначение |
|-----------|------------|
| `create_task` | Создание задачи с проверками |
| `change_task_status` | Изменение статуса с машиной состояний |
| `assign_task` | Назначение исполнителя |
| `add_comment_to_task` | Добавление комментария |
| `get_tasks_by_user` | Получение задач по роли |
| `get_project_statistics` | Статистика по проекту |
| `register_user` | Регистрация пользователя (админ) |
| `fire_user` | Увольнение (мягкое удаление) |
| `restore_user` | Восстановление уволенного |
| `create_project_with_manager` | Создание проекта |
| `change_project_status` | Изменение статуса проекта |
| `get_task_details` | Детальная информация о задаче |
| `get_project_details` | Детальная информация о проекте |

### Триггеры

| Триггер | Событие | Действие |
|---------|---------|----------|
| `update_updated_at_column` | UPDATE на `tasks` | Автообновление `updated_at` |
| `check_task_due_date` | INSERT/UPDATE на `tasks` | Запрет срока раньше текущей даты |
| `log_task_status_change` | UPDATE на `tasks` | Запись в `task_status_history` |
